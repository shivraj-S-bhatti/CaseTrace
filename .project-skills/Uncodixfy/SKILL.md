# Uncodixfy

This project-local copy exists so CaseTrace can follow the same design rules without changing the user's global Codex setup.

## Core rule

If a UI choice feels like a default AI-generated dashboard move, do not use it. Pick the cleaner, more ordinary, product-shaped option instead.

## Keep it normal

- Use straightforward application layouts with fixed rails, top bars, and clearly separated work areas.
- Use solid surfaces, thin borders, restrained radii, and minimal shadows.
- Use readable sans-serif typography with clear hierarchy.
- Use standard tabs, buttons, tables, lists, and panels.
- Keep color calm and functional.
- Let information hierarchy create interest instead of decorative effects.

## Hard no

- No hero sections inside the application shell.
- No eyebrow labels or decorative uppercase captions.
- No glassmorphism, blur haze, or floating premium shells.
- No oversized rounded corners.
- No serif headline plus sans-serif body combo.
- No gradient-heavy backgrounds.
- No pill overload.
- No KPI-card grid as the default first move.
- No fake charts or filler visuals.
- No copy that explains the UI instead of doing the work.

## CaseTrace translation

- The queue should look like a real analyst queue, not a marketing card stack.
- The active case should open straight into the operational workspace.
- Recommendation, policy checks, and audit packet should stay visible without decorative toggles.
- Validation should read like a compact readiness report, not a showcase section.
- Use normal product spacing and restrained motion only where it helps orientation.
